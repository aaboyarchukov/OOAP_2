# Замыкание иерархии

В данном уроке мы рассмотрим механизм замыкания иерархии.

### Замыкание иерархии
---
При проектировании иерархии с множественным наследованием, желательно определить не только сверху "супер-родителя", но и замкнуть нашу иерархию снизу (**добавить пустой закрытый класс None, потомок всех классов**). С помощью такого механизма, мы сохраняем полную систему типов в нашем проекте.
Все классы-листья становятся прямыми предками `None`, а `Void` — это глобальный объект типа `None`, который можно подставлять в переменную любого типа. Именно потому что `None` является потомком всех классов — а значит подходит под любой тип в иерархии.

Таким образом мы избавляемся от использования `null` значений в разных языках, и полностью переходим на модель типов (Null Object Pattern).

***Но важно помнить что это не убирает проблему полностью — вызов метода на `Void` всё равно приведёт к исключению в рантайме. Просто мы остаёмся в рамках чистой системы типов проекта.***

### Рефлексия
---
Эталонное решение:

**Java:**

```java
class General implements Serializable {
   ...
}

class Any extends General { }

...

final class None extends Any /*A, B, ....*/ { }

//

class Test {
    public static Any getSome() {
        return new None();
    }

    public static void setSome(Any any) {
        if (any instanceof None) {
            System.out.println("wrong value!!!");
        }
    }
}
```

**Python:**

```python
В Python нету проверок типов, поэтому можно создать любой класс без предков/потомков, не входящий в нашу систему типов, унаследованную от General.
```

Проанализировав эталонное решение понял, что иерархию построил верно, но не учел того, что в Python не встроена проверка типов, что говорит о том, что мы можем просто сделать класс `None`, который будет унаследован от `General`, и не будет необходимости использовать множественное наследование и замыкание.
