# Политика скрытия методов при наследовании

В данном занятии мы рассмотрим какие есть виды доступности методов при наследовании, а также посмотрим какие из них реализуются в выбранном ЯП.

### Наследование и видимость методов?
---
Между ними принципиальная разница, **наследование** - является механизмом, который выстраивает отношения между классами в иерархии, а **видимость методов** - про отношение между классом и пользователем этого класса, то есть класс предоставляет определенный интерфейс, который может использоваться пользователем.
### Доступность методов
---
При наследовании можно сделать определенный уровень видимости, который будет влиять на использование методов родителя в наследниках.

Примем следующие 4 вида:
1. метод публичен в родительском классе А и публичен в его потомке B;  
2. метод публичен в родительском классе А и скрыт в его потомке B;  
3. метод скрыт в родительском классе А и публичен в его потомке B;  
4. метод скрыт в родительском классе А и скрыт в его потомке B.

### Чем является наследование
---
Прежде всего наследование должно строится не от технических возможностей и тонкостей, а от спецификации, правильной теории, методологической и формальной основы, то есть мы отталкиваемся прежде всего от того, что описали в спецификации.

**Тогда простой вывод о том, что доступными остаются только те методы, которые открыты в спецификации!**

### Примеры
---

#### Python

Вообще в `python` нельзя построить такую иерархию с доступами, только условную, которая основывается на нейминге методов.

[example.py](https://github.com/aaboyarchukov/OOAP_2/blob/main/lesson_13/example.py)

```python
class A:
    def __init__(self, class_type):
        self.type = class_type

    # 1. метод публичен в родительском классе А и публичен в его потомке B    
    def get_type(self):
        return self.type

    # 4. метод скрыт в родительском классе А и скрыт в его потомке B
    def __inner_method_for_A(self):
        pass
    
    # 2. метод публичен в родительском классе А и скрыт в его потомке B
    def _outer_method_for_A_not_for_B(self):
        pass

class B(A):
    def __init__(self, class_type):
        super().__init__(class_type)
        pass

```

#### Java

```java
public class A {
	// ...
	// 1. метод публичен в родительском классе А и публичен в его потомке B
	public void method1() {}
	
	// 4. метод скрыт в родительском классе А и скрыт в его потомке B
	private void method4() {}
	
	// 2. метод публичен в родительском классе А и скрыт в его потомке B
	protected void method2() {}
}


public class B extends A {
}
```
