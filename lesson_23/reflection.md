# Типичные проблемные особенности наследования

В типичные проблемы наследования входят:
- **усиление предусловий** - Так, например, при реализации `BoundedStack` мы усиливаем предусловие операции `push` так как стек ограничен размером, тем самым мы не сможем пользоваться оригинальным методом и нам придется его переопределять, что не очень хорошо, ведь по сути он делает все то же самое. Казалось бы, что можно добавить через композицию, но тогда иерархия рушится логически.

- **ослабление постусловий** - То есть ослабленные условия, не могут гарантировать выполнение интерфейса предка, так например, метод `clear` изначально удаляет N элементов, а в потомке удаляет строго определенную часть, например только 2 элемента, тогда идет нарушение конртакта.

Можно обойти такую проблему несколькими способами:
- метод, который контроллирует усиление/ослабление (метод `full` для `push`) - абстрактное предусловие
- наследование универсального стека от BoundedStack
Как видно, универсального метода нет, что говори об ограниченности ООП

Таким образом мы допускаем только следующее:
- ослабление предусловий
- усиление постусловий

Эти проблемы выражены в [[LSP|LSP]], данный принцип гласит о том, что нельзя ломать логику родительских методов. То есть потомки не должны ломать базовую логику предков и при этом предоставлять полный доступ к родительскому интерфейсу.

Вывод: потомок принимает шире, гарантирует строже. Вызывающий код ожидает контракт предка - потомок должен его выполнить в любой ситуации.
### Пример
---
В пример можно также привести некоторые структуры данных, например универсальный родительский класс Tree, от котрого мы наследуем дерево ограниченной высоты, мы не можем усилить предусловие метода `add` тем, что высота ограничена, тогда как в универсальном родительском классе, она никак не ограничена.

```python
class Tree:
	def __init__(self, root):
		self.root = root
	# предусловие: у дерева есть корень
	# постусловие: добавлен новый узел
	def add(self, node): ...
	
class BoundedHeightTree(Tree):
	def __init__(self, root, height):
		super().__init__(root)
		self.height = height
	
	# предусловие: у дерева есть корень и высота дерева не превышает лимит
	# (усиление предусловия) 
	# постусловие: добавлен новый узел
	def add(self, node): ...

```

Есть родительский класс Дозатор, у которого есть метод `get()`, с помощью которого выдается содержимое дозатора, у него единственное предусловие - наличие содержимого в отсеке. Мы наследуем СенсорныйДозатор, у которого усиливается предусловие тем, что он требует наличие рук на определенном расстоянии. В таком случае мы не везде можем использовать СенсорныйДозатор как Дозатор везде - нарушаем LSP.
